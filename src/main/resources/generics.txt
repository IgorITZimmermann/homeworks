Generics обобщения
Механизм для написания обобщенного кода, который может работать с любыми типами данных,
при сохранении правил строгой типизации
Бывает в классах, интерфейсах, методах

ArrayList<E> где <E> обозначает обобщенный тип данных, тип элементов, которые будут и могут храниться в списке.
ArrayList<String> список работает только со строками

Преимущества
1. Безопасность. Компилятор контролирует допустимый тип данных, что снижает риск возникновения ошибок при добавлении данных
2. Отсутствие кастомизации, методы возвращают строго указанный тип данных
3. Улучшенная читаемость. ArrayList<String> внешне указывает на тот тип данных, котрые ожидаются в листе.

Для создания обобщенного класса использовать угловые скобки < > и внутри указать тип параметра <T>
T это параметр типа данных

Ограничения
<T extends Number> ограничения на тип значений. Тип должен быть либо Number либо наследником от Number
-->Integer, Double, Float... BoxNumber<String> не скомпилируется
